⭐ 推薦給大多數人

用 Claude Desktop 的同仁適用 — 下載一個檔案、雙擊、Claude 就會問你要不要啟用。整個過程 3 分鐘。

📦

一鍵安裝包（.mcpb）

把這個檔案下載到電腦，雙擊就能裝進 Claude Desktop。

  1. 點下方按鈕下載 iso-form-mcp.mcpb
  2. 雙擊那個檔案
  3. Claude Desktop 跳出「要加入這個工具嗎？」→ 點確認
  4. 之後在 Claude 直接問「幫我查 ISO 9001 第 4 條」就會用到
⬇️ 下載安裝包（2.3 MB）

給技術同仁用

用 Cursor、VS Code、Cline、n8n 或其他支援 MCP 的工具，用下面任一方式接。

🧰 用 npm 安裝

把這段貼到你的 MCP 設定檔（claude_desktop_config.json 或 .mcp.json）：

{ "mcpServers": { "iso-form": { "command": "npx", "args": ["-y", "@simhope/iso-form-mcp"] } } }

📂 從原始碼 clone

git clone https://gitlab.simhope.local/internal/iso-form-mcp.git

📖 為什麼做這個工具

（這裡顯示原本的 rich block editor 內容 — 文字 block、步驟 block、圖片 block、提示 block 等）

內容區塊省略 — 維持現有 block editor，作者可以自由排版加圖
📍 畫面 3 — 提交工具表單（4 欄極簡）
原本 12+ 欄位 → 砍到 4 個必填。其他細節（npm 套件名、.mcpb URL、進階說明）由經企室審核時討論補完。
變動重點：
  • 必填只剩：名字、一句話介紹、主連結、類型
  • 類型每個選項配 「適合什麼情況」說明，非技術同仁也看得懂
  • 進階欄位（部門、folder、適用場景、tag）→ 砍掉，由 AI 從 desc 推測 + 審核時補
  • 「AI 幫我寫文案」按鈕保留 — 還是可以用 Gemini 自動生 tagline / desc

📐 後端欄位設定草案（給程式參考，不會給同仁看）
跟 v1 的差別：取消「多選 deliveryMethods」設計，回到單一 type 欄位（但選項從 3 個 → 5 個）。MCP 的多種安裝方式（npm / mcpb / clone）由 類型特定欄位承接，作者/admin 在審核後補。 
{
  // === 提交時必填（4 個欄位） ===
  "title": "ISO 表單查詢 MCP",
  "tagline": "把 ISO 表單接到 Claude/Cursor",
  "url": "https://github.com/simhope/iso-form-mcp",    // 主連結
  "type": "mcp",   // webapp / download / doc / mcp / api

  // === 系統自動填 ===
  "status": "pending",          // 送出時固定 pending，admin 審後改 live/beta/dev
  "authorUid": "...",
  "createdAt": ..., "updatedAt": ...,

  // === 審核後 admin / 作者補 ===
  "typeData": {
    // MCP 類型專屬欄位
    "mcpbUrl": "https://.../iso-form.mcpb",
    "npmPackage": "@simhope/iso-form-mcp",
    "configSnippet": "{ ... }"
  },
  "dept": "admin",
  "folder": "行政作業專案",
  "scenarios": ["行政簽核"],
  "icon": "📋",
  "color": "c5",
  "blog": { "summary": "...", "blocks": [...] }    // 詳細說明 tab 的內容
}

不同 type 各自的 typeData 欄位

typetypeData 包含卡片 CTA
webapp（無，url 就夠）「🌐 馬上打開」→ 開 url
downloadplatform (win/mac), version, size「⬇️ 下載安裝檔」→ 開 url
docfileType (pdf/docx/xlsx/zip), size「⬇️ 下載」→ 開 url
mcpmcpbUrl, npmPackage, configSnippet「📦 安裝到 Claude / Cursor」→ 開詳情頁 tab 1
apiendpoint, docsUrl, openapiUrl「🔗 看 API 文件」→ 開 docsUrl

一次性資料轉換（migration）

  • 現有 14 筆 tools 的 type 從 webapp / download / showcase → webapp / download
  • 原本 type: 'showcase' 的（目前只有 t9 MasterCAM）改 type: 'download' + 保持 status: 'dev'，url 留空
  • typeData 全部建立成 {}，之後逐筆補
  • 寫一個 scripts/migrate-types.mjs，用 service account 跑一次
