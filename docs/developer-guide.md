# 開發者上架指南

> 適用對象：持有 `developer` 帳號的工具開發者

---

## 一、取得開發者帳號

開發者帳號需由管理員在後台建立，不能自行註冊。

請聯絡 Jason（jasonlin@simhope.com.tw）提供：
- 你的 simhope email
- 你想提交的工具名稱

---

## 二、登入

開啟工具箱首頁 → 右上角 **「👨‍💻 開發者登入」** → 輸入帳號密碼 → 登入後出現 **「🛠️ 我的工具」** → 點擊進入儀表板 `/dashboard`。

---

## 三、提交新工具

在 Dashboard 頁面填寫工具資料：

| 欄位 | 說明 | 必填 |
|------|------|------|
| 工具名稱 | 簡短中文名，6–12 字 | ✓ |
| 副標題 | 一句話吸引人的說明，10–25 字 | ✓ |
| Icon | 單一 Emoji，預設 📦 | ✓ |
| 類型 | 網頁版 / 下載版 / 展示 | ✓ |
| 部門 | 主要適用部門 | ✓ |
| 功能說明 | 詳細說明，50–80 字，對非技術人員友善 | ✓ |
| 工具連結 URL | 網頁版填連結；下載版填下載頁或 Drive 連結；展示型可留空 | |
| 使用步驟 | 最多三步，動詞開頭，每步 8 字內 | |
| 標籤 | 關鍵字，逗號分隔 | |

### 使用 AI 文案生成

覺得想不到怎麼寫文案？點表單上方的 **「🪄 AI 文案助手」**：
1. 輸入一句話描述你的工具（例如：「幫同仁把大量 PDF 中的空白頁一次刪掉」）
2. 點 **「生成」**，Gemini 2.5 Flash 會自動產生 icon、名稱、副標題、說明、步驟、標籤
3. 確認內容後，點 **「填入表單」** 即可套用

> AI 生成的文案可能需要微調，務必確認內容正確再送出。

### 送出

資料填好後點 **「送出工具」**，系統會顯示「已送出！經企室審核ok後即上架」，等管理員審核通過即自動上架。

## 四、編輯工具說明書（Block Editor）

說明書是工具詳情頁的核心內容，讓同仁能快速了解如何使用。

### 進入編輯模式

工具詳情頁（`/tool/{id}`）右上角點 **「✏️ 進入編輯」**（需為該工具的作者或 admin）。

### Block 類型

| Block | 用途 |
|-------|------|
| 📝 文字 | 段落說明，支援 `**粗體**`、`## 標題`、`- 清單`、`` `code` `` |
| 📋 步驟 | 自動編號的步驟列表，每行一步 |
| 🖼️ 圖片 | 貼上截圖 URL，**務必填寫說明文字（即圖片 alt）** |
| 🎬 影片 | 貼 YouTube 連結 |
| 💡 提示 | 綠底小提示框 |
| ⚠️ 警告 | 黃底警告框 |

### 建議最少補齊

1. **1 個文字 block**：工具背景與適用情境
2. **1 個步驟 block**：完整使用步驟
3. **1 個圖片 block + 說明文字**：畫面截圖（沒有截圖就沒有 alt，會影響無障礙性）

### 新增截圖

1. 截圖存為 PNG/JPG
2. 上傳到 Google Drive / Firebase Storage / Imgur 等，取得公開連結
3. 在 block editor 新增「圖片」block，貼上連結
4. **說明文字欄位一定要填**（這也是圖片的 alt 文字）

### 儲存

右上角點 **「💾 儲存」**，顯示「儲存成功！」即完成。若顯示「儲存失敗：你沒有編輯此工具的權限」，請確認是否已登入且為該工具作者。

## 五、工具狀態說明

你提交的工具狀態流程：

```
pending（待驗收）→ live（使用中）
                 → dev（開發中）  ← 管理員視需要調整
                 → terminated（已終止）
```

狀態由管理員在後台調整，你可在 Dashboard「我的工具」區塊查看目前狀態。
