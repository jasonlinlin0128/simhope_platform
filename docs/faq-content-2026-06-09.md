## 🔐 login — 登入 / 帳號

### [order 1] 我要怎麼登入？

用**公司 Google 帳號**一鍵登入最快；也可以用 **Face ID / 指紋（passkey）** 免密碼登入。少數由經企室建立的開發者帳號可用 Email + 密碼。右上角「登入 / 註冊」即可開始。

### [order 2] 第一次登入要做什麼設定嗎？

不用。第一次用 Google 登入會**自動建立你的「一般同仁」帳號**，馬上就能瀏覽與使用所有公開工具。想上架工具或取得更多權限，再另外申請開發者（見「提需求 / 上架」分類）。

### [order 3] 什麼是 Face ID / 指紋登入（passkey）？要怎麼設定？

passkey 讓你**免打密碼、免選 Google 帳號**，直接用裝置的 Face ID / 指紋進來。設定方式：登入後到「我的工具 / 儀表板 → 安全設定 → 註冊這台裝置」。每台裝置各註冊一次即可；私鑰只留在你的裝置上，平台不會拿到。

### [order 4] 我登入後看不到「上架工具 / 管理後台」？

這是**權限分級**：登入後每位同仁右上角都有「**我的工具**」（上架表單與安全設定都在這），但**實際送出上架需要開發者權限**——一般同仁送出會被系統擋下，請先申請開發者（見「提需求 / 上架」）。「**管理後台**」只有經企室（管理員）帳號會出現。完整權限說明見「取用說明」頁。

### [order 5] 登入有問題 / 登不進去怎麼辦？

先確認用的是**公司 Google 帳號**。若你原本是純 Google 帳號、沒有密碼，請勿走密碼登入。仍有問題請聯絡經企室協助。

## 🧭 usage — 使用工具

### [order 1] 我要怎麼找到需要的工具？

三種方式：① **首頁**依類別（工具 / 平臺 / 專案 / MCP / Skill）瀏覽，或看「痛點解法」找跟你情境相符的；② 到**資源中心**用搜尋框打關鍵字（名稱 / 描述 / 標籤都能搜）；③ 從首頁痛點卡直接點進對應工具。

### [order 2] 工具要怎麼開始用？

看工具的**類型**：🌐 網頁應用點「馬上打開」即可用、免安裝；⬇️ 軟體下載點「下載安裝檔」；🔌 MCP / 🧠 Skill 需安裝到你的 AI（見「MCP / Skill 怎麼裝」）；📍 場域工具是綁定特定設備 / 場地的（如影印機、產線電腦）。

### [order 3] 工具上的「使用中 / 測試中 / 開發中」是什麼意思？

工具狀態標籤：**使用中**＝穩定可正式用；**測試中**＝可用但仍在收回饋；**新上線**＝剛發布；**開發中**＝規劃 / 製作中，先讓你知道在做；**已終止**＝不再維護（保留紀錄）。

### [order 4] 我點工具連結卻看到「找不到這個工具」？

代表這個工具**不存在、已下架，或你目前沒有檢視權限**（例如還在審核中、非你提交的草稿）。可點頁面上的「回資源中心」重新瀏覽現有工具；若你確定該工具應該存在，請聯絡經企室。

### [order 5] 工具卡上的類型圖示（🌐 / ⬇️ / 🔌 …）代表什麼？

標示這個資源的使用方式：🌐 網頁應用（瀏覽器直接開）、⬇️ 軟體下載（裝到電腦）、🔌 AI 連接器（MCP，串到 Claude / Cursor）、🧠 Skill（Agent 技能）、📍 場域工具（綁定設備 / 場地）、📄 文件 / 表單、🧩 API / SDK 等。

## 📤 submit — 提需求 / 上架

### [order 1] 我做了一個工具，想上架到平台，怎麼做？

先**申請成為開發者**（見下一題）。通過後，到「上架工具」儀表板填 **4 個必填**（名字、一句話介紹、連結、類別）就能送出；其餘細節（截圖、使用步驟、適用部門、進階安裝）由經企室在審核時跟你討論補上，通過後上架。

### [order 2] 怎麼申請成為開發者？

登入後到「**取用說明**」頁點「📩 申請成為開發者」，或在任何頁面用「提需求」送出申請並說明用途。經企室核准後，你的帳號就會升級為開發者，看得到「上架工具」儀表板。

### [order 3] 送出工具後多久會上架？流程是什麼？

送出後工具會進入**待驗收**狀態（此時只有你和經企室看得到）。經企室會跟你討論並補完細節、設定狀態（測試中 / 使用中等），確認後才對全公司公開。時間視內容多寡而定，通常會主動跟你聯繫。

### [order 4] 我只是「有個想法 / 痛點」，不會自己做，可以嗎？

可以！用任何頁面的「**提需求**」（或首頁的「說出來，我來想辦法」）把你的情境寫下來送出。經企室會評估——不保證每項都能做，但**每一條都會看**，有現成工具會幫你找、沒有就評估開發。

### [order 5] 上架表單裡的「AI 輔助」是什麼？

填表時可用 AI 依你給的「做什麼 / 給誰 / 解決什麼痛點 / 主要功能」自動生成名稱與一句話介紹草稿，省去想文案的時間；生成後你仍可自由修改。送出後經企室審核時也會用 AI 協助補完內容。

## 🛡️ security — 資料安全

### [order 1] 我的登入憑證 / 密碼安全嗎？

登入由 **Firebase Authentication** 處理，平台不會看到也不會儲存你的 Google 密碼。passkey（Face ID / 指紋）的**私鑰只留在你的裝置**，平台只存公鑰、無法反推。整體採業界標準機制。

### [order 2] 我還在審核中 / 草稿的工具，別人看得到嗎？

**看不到。** 未公開（待審核）的工具只有**作者本人和經企室**能看到；其他人即使知道連結也讀不到內容（資料層已限制）。工具通過審核、設為公開狀態後才會對全公司顯示。

### [order 3] 這個平台可以分享給外部 / 廠商嗎？

**不行。** 這是公司**內部使用**的 AI 資源中心，未經授權禁止對外散佈內容或連結。對外需求請先經經企室評估。

### [order 4] 為什麼有些功能要再次驗證 / 限制操作頻率？

為了防止濫用與保護資料，部分動作（如登入、AI 生成）有合理的頻率限制；正常使用不會遇到。若短時間大量操作被擋下，稍等一下再試即可。

## 🧠 install — MCP / Skill 怎麼裝

### [order 1] MCP 工具要怎麼裝到 Claude / Cursor？

用 **Claude Desktop** 的同仁最簡單：到該 MCP 工具的**詳情頁 → 「快速安裝」**下載安裝包（.mcpb），雙擊後 Claude 會問你要不要啟用，整個過程約 3 分鐘。用 **Claude Code / Cursor**（或該工具沒提供 .mcpb）的，改看詳情頁「**進階設定**」，用 npm / config 片段方式加入。詳細步驟見「新手上路」頁的「② 把工具裝進我的 AI」。

### [order 2] Skill（Agent 技能）怎麼安裝？

到該 Skill 的詳情頁**下載 .zip**，解壓到 `~/.claude/skills/` 目錄下，重開 Claude 就會載入。之後在對話中即可使用該技能。

### [order 3] 下載類（⬇️ 軟體）的工具怎麼用？

到詳情頁點「**下載安裝檔**」取得檔案，依工具說明安裝到你的電腦。多數內網桌面工具不需網路即可使用。

### [order 4] 安裝 MCP 前要先準備什麼？

需要一個支援 MCP 的 AI 客戶端：**Claude Desktop**、**Claude Code** 或 **Cursor**。裝好其中一個後，再依詳情頁的「快速安裝」把 MCP 連接器加進去即可。需要程式串接（API）的進階用法見「新手上路」頁第 ③ 段。
