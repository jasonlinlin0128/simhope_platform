## Task 1: #38 dashboard exhaustive-deps（清 1 lint warning）

**Files:** Modify `app/dashboard/page.jsx`

**問題**：`useEffect`（L80）呼叫 `fetchMyTools`，但 `fetchMyTools`（L88，定義在 effect 之後）未列入 deps → `react-hooks/exhaustive-deps` warning（3 個剩餘 warning 之一）+ 潛在 stale-closure。

- [ ] **Step 1: import 加 useCallback（L3）**

old：

```jsx
import { useState, useEffect } from "react";
```

new：

```jsx
import { useState, useEffect, useCallback } from "react";
```

- [ ] **Step 2: 把 fetchMyTools 移到 useEffect 前 + 包 useCallback，effect 加進 deps（L80-101 整段取代）**

> 必須把 `fetchMyTools` 移到 `useEffect` **之前**——否則 deps 陣列 `[..., fetchMyTools]` 在 const 宣告前求值會 TDZ。

old：

```jsx
useEffect(() => {
  if (!authLoading && !user) {
    router.push("/");
  } else if (user) {
    fetchMyTools();
  }
}, [user, authLoading, router]);

const fetchMyTools = async () => {
  setLoading(true);
  try {
    const q = query(
      collection(db, "tools"),
      where("authorUid", "==", user.uid),
    );
    const snap = await getDocs(q);
    setMyTools(snap.docs.map((d) => ({ id: d.id, ...d.data() })));
  } catch (error) {
    console.error("Failed to load tools", error);
  }
  setLoading(false);
};
```

new：

```jsx
const fetchMyTools = useCallback(async () => {
  setLoading(true);
  try {
    const q = query(
      collection(db, "tools"),
      where("authorUid", "==", user.uid),
    );
    const snap = await getDocs(q);
    setMyTools(snap.docs.map((d) => ({ id: d.id, ...d.data() })));
  } catch (error) {
    console.error("Failed to load tools", error);
  }
  setLoading(false);
}, [user]);

useEffect(() => {
  if (!authLoading && !user) {
    router.push("/");
  } else if (user) {
    fetchMyTools();
  }
}, [user, authLoading, router, fetchMyTools]);
```

> `fetchMyTools` 用 `user.uid` → useCallback dep `[user]`（只在 user 變時重建）。effect deps 加 fetchMyTools（穩定 per user）→ 行為不變（仍只在 user/authLoading 變時跑），但 lint-clean。`handleFormSubmit`（後面）呼叫 fetchMyTools 仍正常（已定義在前）。

## Task 3: #24 enrich-tool GitHub regex 錨定 host

**Files:** Modify `app/api/admin/enrich-tool/route.js`

**問題**：`url.match(/github\.com\/([^/]+)\/([^/?#]+)/)`（L31）未錨定 → `https://evil.com/github.com/x/y` 也 match（雖 fetch target 是 hardcoded api.github.com、無真 SSRF，但 host 解析不精準）。

- [ ] **Step 1: 錨定到 host 開頭（L31）**

old：

```js
const gh = url.match(/github\.com\/([^/]+)\/([^/?#]+)/);
```

new：

```js
const gh = url.match(/^https?:\/\/(?:www\.)?github\.com\/([^/]+)\/([^/?#]+)/);
```

> `^https?://(www.)?github.com/` 錨定協定+host；`(?:...)` 非捕獲 → gh[1]=owner、gh[2]=repo 位置不變。擋掉 `evil.com/github.com/...` 誤判。

- [ ] **Step 2: build** → `npm run build` 綠（route 編譯無誤）。

- [ ] **Step 3: commit**

```bash
git add app/api/admin/enrich-tool/route.js
git commit -m "fix(enrich-tool): GitHub URL regex 錨定 host 防誤判 (audit #24)

原 regex 未錨定，evil.com/github.com/x/y 也會 match（fetch target 雖
hardcoded api.github.com 無真 SSRF，但 host 解析不精準）。錨定到
^https?://(www.)?github.com/，capturing group 位置不變。

Co-Authored-By: Jason simhope ai agent <jasonlin@simhope.com.tw>"
```
